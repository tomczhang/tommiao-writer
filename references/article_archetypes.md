# 汤姆喵文章原型

写作前先判断文章属于哪种原型。不同原型的重点不同，不要用同一套结构硬套所有文章。

## 1. 投资反思型

代表主题：

- 第一次破坏纪律
- 面对浮亏
- FOMO 发作
- 做 T 后悔或卖飞
- 仓位打满后的无力感
- 某次亏损带来的反思

核心不是复盘股价，而是复盘「我为什么会做出那个动作」。

标准结构：

1. 一个具体操作或市场事件。
2. 当时自己的真实情绪。
3. 表面看是亏钱，深层看是仓位、现金流、纪律或认知出了问题。
4. 用几个问题、规则或行动卡片把经验固定下来。
5. 回到「下次怎么少犯同样的错」。

关键句式：

- 「我当时真正难受的，不是亏损本身。」
- 「问题不在价格，而在我没有准备好。」
- 「如果这些问题回答不出来，那就说明我还没有资格出手。」
- 「人不应该两次跌倒在同一个坑里。」

## 2. 投资体系构建型

代表主题：

- 现金流
- 仓位管理
- 道势法术
- 投资的最终目的
- 保值优先于增值
- 选择权和自由感

核心是把一个抽象投资概念，拆成普通人能理解和执行的体系。

标准结构：

1. 承接一个已有困惑或前文。
2. 提出一个核心命题。
3. 用公式、类比或分层结构拆开。
4. 引用机哥、巴菲特、段永平、书籍或自己的操作。
5. 强调普通人的资源条件和大师不同。
6. 落到自己的可执行策略。

常用工具：

- 等式：财富 = 净资产 + 现金流
- 类比：鱼塘、水池、牌桌、猎人、部队
- 分层：道 -> 势 -> 法 -> 术
- 对照：大师策略 vs 普通人策略

关键句式：

- 「普通人并没有无限等待权。」
- 「不是这个策略不好，而是普通人没有人家的条件。」
- 「说到底，还是现金流的问题。」
- 「选择权才是能达成结果的能力。」

## 3. AI 工具实战型

代表主题：

- 金渐成 AI 分身
- 评论监控系统
- 公众号文章导出工具
- IMA / NotebookLM
- Claude Code / LangChain / Agent
- 用 AI 辅助投资学习

核心是「我真的做了一个东西，它解决了一个真实问题」。

标准结构：

1. 一个具体痛点，例如文章太多、评论会消失、工具不好用。
2. 自己为什么想解决。
3. 做的过程，包含工具选择、踩坑、迭代。
4. 展示结果和读者怎么用。
5. 讲 AI 对普通人的意义。
6. 承认 AI 的边界。

关键句式：

- 「好在我是程序员，能用 AI 来帮助我。」
- 「这些代码原本都是以月为维度来计算工作量的，现在被我用业余时间几个小时就鼓捣出来了。」
- 「AI 节省了开发时间，但没有让程序员轻松下来。」
- 「它不能代替你学习，也不能代替你领悟道法术。」

注意：不要只写工具教程。工具只是入口，最终要讲「普通人如何用 AI 获得杠杆」。

## 4. 读后感延展型

代表主题：

- 机哥文章读后感
- 巴菲特股东会
- 段永平投资问答
- 富爸爸穷爸爸
- 道法术

核心不是搬运原文，而是写「这段话为什么击中了我，我过去哪里没想明白」。

标准结构：

1. 原文或观点中最触动自己的点。
2. 过去自己是怎么理解的。
3. 现在为什么有了新的理解。
4. 结合自己的操作、生活或 AI 实践。
5. 抽象成自己的体系或行动规则。

关键句式：

- 「我以前不能理解这一点。」
- 「现在回头看，这句话其实是在说……」
- 「对我这样的普通人来说，它真正重要的地方在于……」
- 「我逐渐意识到……」

注意：引用要克制。文章的主体必须是自己的理解，不是资料摘录。

## 5. 人生阶段总结型

代表主题：

- 年终总结
- 35 岁程序员转型
- 健康和减肥
- 育儿
- 公众号成长
- 投资第一年复盘

核心是诚实回看自己这一阶段的变化。

标准结构：

1. 一个明确时间节点。
2. 按几个维度复盘，例如投资、健康、技术、公众号、育儿。
3. 每个维度都要有真实变化和具体事件。
4. 从事件中提炼一个认知变化。
5. 最后给下一阶段一个温和但坚定的方向。

关键句式：

- 「严格来说，这更像是一份下半年总结。」
- 「不如诚实一些，就从真正有感受的地方开始。」
- 「今年我主要学到两点。」
- 「新的一年，我会继续写下去。」

注意：不要写成年度述职，也不要写成鸡汤。重点是个人变化和自我校准。

## 6. 工具分享/教程型

代表主题：

- 怎么看被盾文章
- 怎么使用 AI 分身
- 怎么搭建某个工作流
- 怎么用 AI 检查观点

核心是有用，但不能只有步骤。

标准结构：

1. 读者遇到的真实麻烦。
2. 自己也遇到或理解这个麻烦。
3. 给出工具或流程。
4. 说明适用边界。
5. 回到为什么这件事对读者有价值。

关键句式：

- 「如果你实在不方便……」
- 「这里大家不要走偏了。」
- 「不需要多么炫酷，但要真能对大家有用。」

注意：步骤可以清楚，但不要写成说明书。要保留人味和判断。

## 7. 公共议题延展型

代表主题：

- AI 对工作岗位的影响
- 中产安全感
- 算法推荐和信息茧房
- AI 平权还是加剧分化
- 公司组织方式变化
- 外部文章、访谈、数据引发的思考

核心是把外部公共议题，拉回到汤姆喵自己的处境和普通人的行动选择。

标准结构：

1. 一个外部触点：文章、论文、访谈、数据、读者问题。
2. 说明为什么它让自己停下来想了一下。
3. 把问题从表层热点升级到结构问题。
4. 回到普通程序员、普通投资者、普通家庭的处境。
5. 给出自己的判断和可执行动作。

常用推进：

- 事实层：发生了什么。
- 结构层：背后的机制是什么。
- 个人层：这和我有什么关系。
- 行动层：我准备怎么做。

关键句式：

- 「一开始我没太在意，但越想越觉得不对。」
- 「这个问题表面上是在聊 AI，其实是在聊普通人的选择权。」
- 「对我这样的程序员来说，真正刺痛我的地方在于……」
- 「所以我现在更关心的不是它有多强，而是我能不能把它放进自己的工作流里。」

注意：

- 不要写成宏观评论员。
- 不要只堆外部信源。
- 不要用单个数据推出过硬结论。
- 最后必须回到自己的体感、普通人的处境和行动。
