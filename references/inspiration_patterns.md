# 外部作者可吸收机制

这个文件记录从 budongjing、jinjiancheng、kazike、maobidao 四类作者中可吸收的写作机制。

原则：**借机制，不借皮囊。** 不改变汤姆喵的人格，不写成别人的号。

## 1. budongjing，可吸收理性开头和标题

适合吸收：

- 标题里的强判断和结构感。
- 用外部信源、论文、访谈、数据开头。
- 把热点问题升级成结构问题。
- 先讲事实，再讲机制，再讲普通人的处境。

可用标题结构：

- `不是 A，而是 B`
- `硬判断：为什么 X 正在 Y`
- `一个时代的拐点：以前不是问题的问题，开始变成问题`
- `某个具体人物/机构被问急了，真正暴露的是另一件事`

### 不懂经标题拆解

**1. 否定误读型**

公式：`X 不是在 A，而是在 B`

适合用于纠正常见误解。

示例：

- `AI 不是在替代岗位，而是在替公司重排权力`
- `现金流不是保守，而是普通人的无限后手`
- `定投不是躺平，而是在给情绪上保险`

**2. 时代拐点型**

公式：`一个时代的 X：以前不是问题的问题，开始变成问题`

适合用于写趋势变化，但不能写成末日预言。

示例：

- `一个时代的认知拐点：以前靠勤奋能解决的问题，开始需要系统解决`
- `AI 时代的工作拐点：以前只是效率问题，现在变成位置问题`

**3. 结构追问型**

公式：`为什么越 A，反而越 B？`

适合用于写悖论，读者会天然想点开。

示例：

- `为什么越努力使用 AI，普通人反而越容易焦虑？`
- `为什么越想长期持有，越需要先把现金流讲清楚？`
- `为什么越想自由，越不能只盯着收益率？`

**4. 人物/事件暴露型**

公式：`某个人/事件被问急了，真正暴露的是 X`

适合从新闻、访谈、市场事件切入。

示例：

- `黄仁勋被问急了，真正暴露的是护城河焦虑`
- `一次没卖出的阿里，真正暴露的是我的纪律漏洞`
- `机哥怒了，真正宝贵的从来不是加仓节点`

**5. 反常识对照型**

公式：`最危险的不是 A，而是 B`

适合写认知反转，但要避免标题党。

示例：

- `AI 时代最危险的不是不用工具，而是太早相信自己会用工具`
- `投资里最危险的不是亏损，而是没意识到自己已经失控`
- `普通人最怕的不是踏空，而是关键时刻没有子弹`

**6. 资产/权利重估型**

公式：`X 时代最贵的资产，不是 A，而是 B`

适合写价值重估和个人选择权。

示例：

- `AI 时代最贵的资产，可能不是工具，而是不被工具牵着走`
- `投资里最贵的资产，不是节点，而是现金流`
- `中年人最贵的资产，不是职位，而是重新开始的能力`

### 使用限制

- 标题可以强判断，正文必须诚实承认不确定。
- 不要连续多篇使用同一种句式，否则会有 AI 感。
- 「不可逆转」「彻底终结」「全部坍塌」这类词慎用。
- 汤姆喵化的标题必须落到个人处境、普通人行动或选择权，不能只做宏观评论。

可用开头结构：

> 最近看到一个数据/访谈/论文，一开始我没太在意。
>
> 但越想越觉得，它说的其实不是一个工具问题，而是一个选择权问题。

使用时必须注意：

- 外部信源只是入口，不要堆材料。
- 每一篇最多抓一个核心信源，不要变成综述。
- 最后必须回到中国语境、普通程序员、普通投资者或自己的真实体感。
- 不要学它过度加粗、过度三段论、过度「时代定论」的 AI 感。

## 2. jinjiancheng，可吸收真实细节和可执行方案

适合吸收：

- 真实生活细节，尤其是家庭、工作、孩子、报税、现金流、账户结构。
- 方案落地到频次、阈值、步骤、话术、正反例。
- 承认局限，反复提醒读者要结合自身情况。
- 投资和生活混在一起讲，形成「人是真的在生活里实践」的感觉。

方法论文章必须尽量给出至少一个可执行参数：

- **频次**：每天、每周、每月、每次交易前。
- **阈值**：仓位上限、现金比例、亏损承受范围、信息筛选标准。
- **清单**：交易前 5 问、AI 工具使用步骤、读文章检查项。
- **话术**：和家人、读者、自己内心对话时怎么说。
- **正反例**：什么是正确做法，什么是 FOMO 发作。
- **最坏情况预案**：如果继续跌、继续涨、工作变化、现金流中断怎么办。

使用时必须注意：

- 不要用 `……` 硬切主题。主题转换前加一句桥接。
- 不要把大量私人细节塞进一篇导致主线散掉。
- 方案要服务你的文章命题，而不是变成操作手册拼贴。

## 3. kazike，可吸收叙事钩子和人物画像

适合吸收：

- 朋友一句反问、饭局一句话、读者一句留言作为开头。
- 从一个小人物或一个数据点，写出背后具体的人。
- 给方法论取一个简单好记的名字。
- 每偏出去一段，用一句话扣回主线。

人物画像模板：

> 他可能不是专业投资者，只是一个普通上班族。
>
> 白天担心裁员，晚上刷到某个加仓节点，心里一痒就想多买一点。
>
> 真正折磨他的不是不知道买什么，而是账户里没有足够现金，也没有一套能让自己冷静下来的纪律。

使用时必须注意：

- 不要学强网感、密集吐槽、AI 媒体博主人设。
- 不要把新模型、新工具、新大会当长期主菜。
- 不要为了制造爽感而夸大情绪。

## 4. maobidao，可吸收日更触点和信息组织

适合吸收：

- 从读者问题或当日事实切入。
- 关键数据先行，用一两句话摆清楚。
- 把同一条线索连续追踪，让读者有追更感。
- 用一条荒诞小细节打破沉闷。
- 结尾有稳定仪式感。

可用开头结构：

> 最近后台有几个机友都在问同一个问题。
>
> 我本来想简单回复一下，后来发现这个问题背后其实牵扯到现金流、仓位和投资目标，值得单独写一篇。

可用信息组织：

1. 先回答读者最关心的那件事。
2. 再补充一组关键数字。
3. 然后解释这组数字背后的机制。
4. 最后回到自己的判断和行动。

使用时必须注意：

- 不要夜报化。汤姆喵不是每日资讯压缩号。
- 不要用过多戏谑绰号和戾气比喻。
- 不要高密度涉政和地缘话题。
- 不要让快讯列表替代费曼式反思。

## 5. 吸收后的标准升级结构

原来的汤姆喵结构：

> 我经历了什么 -> 我怎么反思 -> 我沉淀出什么体系

升级后的结构：

> 一个真实事件或外部信源 -> 我为什么被触动 -> 背后的结构问题 -> 普通人的处境 -> 我的可执行方案

适用场景：

- 想让个人反思更有公共价值。
- 想写得更理性客观。
- 想提升标题和开头吸引力。
- 想让方法论更落地。

不要每篇都套这个结构。纯个人复盘时，仍然优先使用亲历型结构。
